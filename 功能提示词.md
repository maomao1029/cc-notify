请为我编写一个名为 `claude-notify` 的桌面通知工具，用于解决 Claude Code 在终端中等待用户确认（Y/n）时，用户因切屏而错过提示的问题。

### 核心功能

1. **包装执行**  
   用户通过 `claude-notify claude`（或任意命令）启动目标进程，工具代理其 stdin/stdout/stderr，用户无感知。

2. **智能检测**  
   实时监控终端输出流，当检测到以下任意确认模式时触发通知：  
   - 包含 `"Y/n"`、`"y/N"`、`"yes/no"`  
   - 包含 `"Confirm"`、`"确认"`、`"Proceed"`、`"Allow"`、`"Apply changes"`  
   - 以 `?` 结尾且后面紧跟输入光标等待的行  
   - 支持通过正则表达式配置文件自定义匹配规则

3. **系统级弹窗 + 按钮交互**  
   - **Windows**：使用原生 Toast 通知，通知内包含 **Yes** 和 **No** 按钮。  
   - **Linux**：优先使用 `notify-send` + 按钮（需要桌面环境支持 action），若不可用则 fallback 到 `zenity` 或 `yad` 对话框（同样提供 Yes/No 按钮）。  
   - 点击按钮后，工具自动向被代理进程的 stdin 写入 `y` 或 `n`，无需切回终端。  
   - 通知设置为**紧急/持久**级别（不会自动消失，需用户点击）。

4. **强提醒**  
   通知同时播放一次短提示音（系统 beep 或自定义音频），且声音开关可配置。

5. **去重机制**  
   同一确认请求（基于输出行前 80 字符）在 **10 秒内**不重复弹窗。

### 技术要求

- **语言**：Python 3.8+（优先，便于跨平台打包）。  
- **平台支持**：Windows 10/11、Linux（Ubuntu/Fedora 主流发行版）。  
   *macOS 暂不要求，可做简单 fallback（仅普通通知，无按钮）。*
- **通知与按钮**：  
  - Windows：使用 `winrt` 或 `plyer` 的 Toast 扩展。  
  - Linux：使用 `notify-send --action` 或调用 `zenity` / `kdialog`。  
- **音频**：Windows 用 `winsound.Beep`，Linux 用 `paplay` 或 `beep`。  
- **配置**：支持 `~/.claude-notify.json`，可自定义：  
  - 正则匹配规则列表（`patterns`）  
  - 忽略的匹配模式（`ignore_patterns`）  
  - 是否开启声音（`sound`）  
  - 去重冷却秒数（`cooldown_seconds`）
