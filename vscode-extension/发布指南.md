# VS Code 扩展发布指南

基于 cc-notify 项目的实际发布经验总结。

## 环境准备

```bash
# 安装 Node.js（如未安装）
sudo apt install nodejs npm

# 安装发布工具
npm install -g ovsx
```

## 第一步：创建命名空间（仅首次）

1. 浏览器打开 https://open-vsx.org/
2. 右上角 Login → 用 GitHub 账号登录
3. 右上角头像 → Settings → Namespaces → Add namespace
4. 填入你的发布者名称（如 `maomao1029`）

## 第二步：获取 Access Token（仅首次）

1. Settings → Access Tokens → Generate new token
2. 描述随便填 → Generate token
3. **立刻复制 token**，只显示一次

## 第三步：登录

```bash
cd 你的扩展目录
ovsx login 你的发布者名 -p 粘贴你的token
```

看到 `🚀 PAT valid to publish at 你的发布者名` 表示登录成功。

## 第四步：发布

每次发布前先改版本号（`package.json` 中的 `version` 字段），不能重复发布同一版本。

```bash
# 改版本号（手动改或用 sed）
sed -i 's/"version": "1.0.X"/"version": "1.0.Y"/' package.json

# 发布
ovsx publish
```

看到 `🚀 Published 你的发布者名.cc-notify v1.0.Y` 表示成功。

## 第五步：同步 GitHub

```bash
git add -A
git commit -m "发布 v1.0.Y：描述改动"
git push
```

## 注意事项

| 问题 | 解决 |
|------|------|
| 没显示 README | `.vscodeignore` 中不要有 `*.md`，且 README.md 要放在扩展目录中 |
| 同名版本重复发布 | 改 `package.json` 中的 `version` 号 |
| 图片不显示 | README 中图片路径要相对于扩展目录（如 `img/xxx.png`） |
| Namespace not found | 先到网站的 Settings 里创建 namespace |
| GitHub 推送超时 | 国内网络问题，重试 `git push` |

## 扩展目录结构参考

```
vscode-extension/
├── package.json          # 扩展清单（名称、版本、命令等）
├── extension.js          # 扩展入口代码
├── .vscodeignore         # 打包时排除的文件
├── README.md             # 扩展主页说明
├── icon.png              # 扩展图标（128x128）
├── img/                  # README 中引用的图片
│   └── screenshot.png
└── wrapper.py            # 扩展依赖的脚本文件
```

## 常用 ovsx 命令

| 命令 | 作用 |
|------|------|
| `ovsx login <name> -p <token>` | 登录 |
| `ovsx publish` | 发布当前目录的扩展 |
| `ovsx ls-publishers` | 列出已登录的发布者 |
